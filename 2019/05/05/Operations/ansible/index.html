Ansible的入门使用

入门了解

Ansible主要作用是用于部署和统一管理远程服务器。还有其他工具许多开源也是用于部署远程服务器。如SaltSack、Puppet等。我们称用于管理其他远程主机的主机称为管理节点。称被管理的主机为远程主机。

  • 安装
  • 连接

我们通常使用SSH来登录管理。

  • 管理的服务器

我们可以将ansible要管理的服务器添加到 /etc/ansible/hosts

并且可以分组管理。

  • 用命令管理主机

更多的指令请参考官网。

  • 用脚本管理主机

ansible使用的脚本是yaml的格式。文件以yml结束。

deploy.yml的内容为：

本质上yaml格式的脚本，就是一个json格式。

yml与json转换工具

  • yaml中关键字的使用：

  • hosts：主机IP，或者分组名或者关键字 all

  • reomte_user：以哪个用户来执行

  • vars：变量，可以自定义(我猜的)

  • tasks：核心关键字，这里面编辑action，每个action调用一个ansible module

  • handers:相当于方法，本身不会执行，在action中触发才会执行。多次触发执行一次。

需要注意的是：vars后可以不用加 - 就声明变量。而其他下下一级的关键字，需要加-

进阶module的使用

常用Module：

  • 调试和测试
    • ping - ping一下远程主机，如果可以通过ansible成功连接，返回pong
    • debug - 用于调试的module，只是简单打印一些消息，有点像linux的echo命令

  • 文件类

    • copy - 从本地拷贝文件到远程节点
    • template - 从本地拷贝文件到远程节点，并进行变量的替换
    • file - 设置文件的属性

  • linux上常用操作

    • user - 管理用户账户
    • yum - red hat系linux上的包管理
    • service - 管理服务
    • firewalld - 管理防火墙中的服务和端口

  • 执行shell命令

    • shell - - 在节点上执行shell命令，支持$HOME和”<”, “>”, “|”, “;” and “&”
    • command - 在远程节点上面执行命令，不支持$HOME和”<”, “>”, “|”, “;” and “&”

常用Module的使用
