Ansible的入门使用

入门了解

Ansible主要作用是用于部署和统一管理远程服务器。还有其他工具许多开源也是用于部署远程服务器。如SaltSack、Puppet等。我们称用于管理其他远程主机的主机称为管理节点。称被管理的主机为远程主机。

  • 安装
  • 连接

我们通常使用SSH来登录管理。

  • 管理的服务器

我们可以将ansible要管理的服务器添加到 /etc/ansible/hosts

并且可以分组管理。

  • 用命令管理主机

更多的指令请参考官网。

  • 用脚本管理主机

ansible使用的脚本是yaml的格式。文件以yml结束。

执行脚本命令：

deploy.yml的内容为：

本质上yaml格式的脚本，就是一个json格式。

yml与json转换工具

  • yaml中关键字的使用：

  • hosts：主机IP，或者分组名或者关键字 all

  • reomte_user：以哪个用户来执行

  • vars：变量，可以自定义(我猜的)

  • tasks：核心关键字，这里面编辑action，每个action调用一个ansible module

  • handers:相当于方法，本身不会执行，在action中触发才会执行。多次触发执行一次。

需要注意的是：vars后可以不用加 - 就声明变量。而其他下下一级的关键字，需要加-

进阶module的使用

常用Module：

  • 调试和测试
    • ping - ping一下远程主机，如果可以通过ansible成功连接，返回pong
    • debug - 用于调试的module，只是简单打印一些消息，有点像linux的echo命令

  • 文件类

    • copy - 从本地拷贝文件到远程节点
    • template - 从本地拷贝文件到远程节点，并进行变量的替换
    • file - 设置文件的属性

  • linux上常用操作

    • user - 管理用户账户
    • yum - red hat系linux上的包管理
    • service - 管理服务
    • firewalld - 管理防火墙中的服务和端口

  • 执行shell命令

    • shell - - 在节点上执行shell命令，支持$HOME和”<”, “>”, “|”, “;” and “&”
    • command - 在远程节点上面执行命令，不支持$HOME和”<”, “>”, “|”, “;” and “&”

分析函数 Lead(),Lag()

判断在一定条件下的两条记录之间的时间差这样的需求。Lead()就是取当前顺序的下一条记录，相对Lag()就是取当前顺序的上一行记录。

语法结构：

参数说明：

  • value_expr 值表达式，通常是字段，也可是是表达式。value_expr本身不支持分析函数，也就是lead不支持多层调用。
  • offset 偏移，应该是很熟悉的数学概念了，或者是相对偏移，表格来开当前行的第offset行，如果offset是整数就表示是顺序下的前第n行，如果是负数就是往后第n行。 如果不提供这个参数，就是默认为1.
  • default 默认值，如果没有找到，应该返回什么值的意思，有点类似nvl(col,value)。如果没有设置，且找不到，那么就返回Null

  • over 理解成在一个结果集范围内，如果后面的partition by为空，那么就是当前的结果集范围内。

  • query_partition_clause 分区语句，对结果集合分区的语句，是可选的，如果没有就是所有的一个分区。

  • Order_by_clause 排序语句 必须需要 ,形如order by xxx desc/asc

设计模式-创建型模式
3 分钟 读完 (大约 403 个字)

前面，我们已经知道了。单例模式和抽象工厂模式。回忆一下。

单例模式就是一个类只有一个实例，而且单例模式的创建分为饿汉和懒汉模式，顾名思义，饿汉就是加载类的时候就创建实例。懒汉就是第一次调用的时候才创建实例。

抽象工厂模式，通过工厂创建实例。通常由很多产生不同对象的具体工厂。而这些工厂由实现同一个抽象工厂。具体工厂生产的产品也可能相同，但是是不同的工厂生产的，可以说是不同的品牌。同时又可以又不同的特性。

英文原话:

Separate the construction of a complex object from its representation so that the same construction process can create different represetations.

将一个复杂对象的构建与他的表示分离,使得同样的构造过程可以创建不同的表.

四种角色:

  • 抽象建造者（Builder）：该角色用于规范产品的各个组成部分，并进行抽象，一般独立于应用程序的逻辑。
  • 具体建造者（Concrete Buider）：实现抽象建造者中定义的方法。并返回一个组件好的产品实例。
  • 产品（Product）：该角色是建造者中的复杂对象，一个系统中会有多个产品类，这些产品类并不一定有共同的接口，可以是完全不想管的
