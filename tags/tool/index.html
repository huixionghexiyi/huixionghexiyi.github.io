git

重要概念；

  • 暂存区

    • 工作区(workspace)
    • 暂存区(stage)
    • 当前版本(master)

指令

初始化：

git init

把文件添加到仓库:三种类型

git add <filename>
git add <filename1> <filename2>
git add .

把文件提交到仓库：message自行修改

git commit -m "message"

查看当前仓库状态：

git status

查看某个修改的内容：

git diff <filename>

查看日志记录：

git log
git log --pretty=oneline

回到前面的版本：

git reset --hard HEAD^
git reset --hard HEAD^^
git reset --hard HEAD~100

查看历史提交的命令(用于查找commit id)：

git reflog

我们使用git add 将修改的文件提交到stage，再使用git commit一次性将stage中的所有提交到master（这里指master这个分支）里。

丢弃工作区的内容：（本质是用版本库替换掉工workspace的内容,也可以看作是切换分支）
若stage中有内容就恢复成stage,没有则恢复master中的。

git checkout -- <filename>

创建并切换分支：

git checkout -b <branch>

将stage中的恢复到workspace中

git reset HEAD <filename>

删除文件

rm <filename>

远程仓库github的设置

创建SSH Key:

ssh-keygen -t rsa -C "meail@mail.com"

再在GitHub里添加用户路径下的 id_rsa.pub公钥。

关联一个远程仓库：

git remote add origin git@server-name:path/repo-name.git

推送master分支到远程仓库：(第一次需要加 -u ，以后不需要)

git push -u origin master

从远程仓库clone 一个仓库：

git clone git@github.com:huixionghexiyi/gitskills.git

删除本地仓库和远程仓库的关联关系：

git remote remove origin

分支

创建分支

git checkout -b dev
或  
git branch dev
git checkout dev

查看分支

git branch

切换分支 

git checkout <branch>

合并分支

将目标分支合并到当前分支
git merge <目标分支>

删除分支

git branch -d <branch>

强制删除

git branch -D <branch>

带参数的查看日志记录

git log --graph --pretty-oneline --abbrev-commit

--graph   表示查看图表
--pretty-oneline 单行查看
--abbrev-commit  缩写的commit id

分支的知识点：

通常情况下：合并分支的时候会默认使用Fast forward 方式。（即文件没有被修改/删除，文件只是被添加（ps:我自己的理解））

禁止Fast forward方式的合并

git merge --no-ff -m "message" <branch>

分支策略：

master应该是非常稳定的，dev是不稳定的，干活的话，就是每个人就从dev上创建自己的分支。

对付bug分支：

stash，储藏(临时保存点)

保存当前工作区的进度

git stash

查看进度的储存list

git stash list

读取进度

git stash apply
git stash drop

或

git stash pop

读取指定的进度

git stash apply stash@{0}

协作

多人需要协作时。在github的项目仓库的设置中的Collaborators选项中将对方邀请进来即可。对方接受，他就对项目可以有写的权限。但是只有项目的拥有者可以选择合并哪个分支到master上（ps：自己的理解）。

标签

创建标签

git tag <标签名>

对历史commit创建标签：

git tag <标签名>  <commit id>

查看标签

git show

创建带说明的标签：

git tag -a <标签名> -m "message" <commit id>

注意，标签总是和commit挂钩。当commit出现在master和dev中那么，这两个分支都可以看见标签。

删除标签：

git tag -d <标签名>

推送标签到远端：

git push origin <tagname>

git push origin --tags

删除远端的标签：

  • 先删除本地的

    git tag -d <标签名>

  • 再删除远端的

    git push origin :refs/tags/<标签名>

使用GitHub

将作者的仓库fork到自己的仓库，然后clone到本地。修改添加。可以pull request 给原作者。

进阶

忽略特殊文件：

只需要有一个.gitignore 文件即可。在里面添加要忽略的文件。

常用用的.gitignore文件：

忽略文件

忽略文件的原则是：

忽略文件的原则是：

  • 忽略操作系统自动生成的文件，比如缩略图等；
  • 忽略编译生成的中间文件、可执行文件等，也就是如果一个文件是通过另一个文件自动生成的，那自动生成的文件就没必要放进版本库，比如Java编译产生的.class文件；
  • 忽略你自己的带有敏感信息的配置文件，比如存放口令的配置文件。

配置别名

将status 取别名为 st
git config --global alias.st status

删除别名：

在config下文件.gitconfig 中有

    $ cat .gitconfig
[alias]
    co = checkout
    ci = commit
    br = branch
    st = status
[user]
    name = Your Name
    email = your@email.com

只需要删除alias中的行即可

总结

每次commit之后都有一个commit id ，而所有的其他操作都是围绕着commit id在运转。所以在项目中commit之前一定要想好。

无论是master、dev、还是HEAD 指针，都是围绕着commit id，变换指向，来实现不同版本之间的切换。
